# ADR-Verzeichnis

## Zweck
Architecture Decision Records dokumentieren relevante Entscheidungen mit Kontext, Alternativen und Auswirkungen.

## Format
- ID (fortlaufend)
- Datum
- Titel
- Status (`proposed`, `accepted`, `superseded`, `deprecated`)
- Kontext, Problem, Entscheidung, Begründung, Alternativen, Auswirkungen, betroffene Bereiche, Risiken/Folgeaufgaben
